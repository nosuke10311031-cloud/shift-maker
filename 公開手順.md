# GitHubで公開する手順（クリックするだけ）

専門用語はカッコ内でかみ砕いています。パソコンで作業してください。

---

## 私（Claude）が済ませた下準備

- このフォルダに「変更履歴をとる仕組み」を付けました（あなたは気にしなくてOK）。
- 公開する画面を `docs` フォルダにまとめました。
- 古いファイルは `archive` フォルダへ移動しました。

ここから先があなたの操作です。

---

## STEP 1. GitHubにログイン（なければ登録）

1. **https://github.com** を開く。
2. アカウントが無ければ「Sign up」でメールアドレスとパスワードを登録（無料）。

## STEP 2. 置き場所を作る（＝「リポジトリ」＝ネット上の保存フォルダ）

1. 右上の「＋」→「**New repository**」。
2. **Repository name** に `shift-maker` と入力。
3. **Public（公開）** を選ぶ。
   - ※公開でも、各自のシフトの中身はコードに入っていないので心配ありません。
4. 緑の「**Create repository**」ボタンを押す。

## STEP 3. ファイルを入れる（ドラッグするだけ）

1. 作成直後の画面にある「**uploading an existing file**」の青いリンクをクリック。
2. パソコンで、この `シフトメーカー` フォルダを開く。
3. 中の **`docs` フォルダ・`README.md`・`公開手順.md` などをまとめて選んで**、
   ブラウザの点線エリアに **ドラッグ＆ドロップ**。
   （`files.zip` は無くてOK）
4. 少し待ってアップロードが終わったら、下の緑「**Commit changes**」（＝「これで保存」）を押す。

## STEP 4. ホームページ機能をオンにする（＝公開）

1. リポジトリ上部の「**Settings**（設定）」を開く。
2. 左メニューの「**Pages**」をクリック。
3. **Source** が「Deploy from a branch」になっているのを確認。
4. **Branch** を「**main**」、フォルダを「**/docs**」に選んで「**Save**」。

## STEP 5. URL（住所）ができる

1. 1分ほど待って、同じ Pages の画面を再読み込み。
2. 上のほうに
   `https://（あなたのユーザー名）.github.io/shift-maker/`
   というURLが出ます。**これがシフトメーカーの住所**。スマホでも開けます。

| 用途 | URL |
|---|---|
| 入口 | `https://（ユーザー名）.github.io/shift-maker/` |
| バイトの希望入力 | `…/shift-maker/staff.html` |
| 管理者用 | `…/shift-maker/admin.html` |
| シフト確認（共有リンクの土台） | `…/shift-maker/view.html` |

---

## STEP 6. 管理者画面に「見る画面の住所」を一度だけ登録

1. `admin.html`（管理者用）を開く。
2. 一番下「7 シフトを共有する」の「**ビューアの公開URL**」欄に
   `https://（ユーザー名）.github.io/shift-maker/view.html` を貼る。
3. これで以後、シフトを作るたびに「共有リンク」が自動でできます。

---

## 毎月の使い方（公開後はこれだけ）

```
バイト：staff.html を開く → 希望を入力 → 「提出コード」を管理者へLINE
管理者：admin.html を開く → コードを取り込む → 自動作成 → 手で微調整
　　　　→「共有リンク」を作って全員へLINE
バイト：リンクを開く → 自分のシフトを確認（名前でしぼり込み）
```

シフトを直したら、**新しいリンクを作って送り直す**だけ。GitHubの再アップロードは不要です。

## 将来コードを直したくなったら

3. もっとラクにしたくなったら、GitHubと自動でつなぐ設定（@claude）も後から足せます。声をかけてください。
